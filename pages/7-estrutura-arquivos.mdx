### Estrutura de Arquivos em Disco

```text {all|2|3|4,24,25|5|6-23|26|all}
~
└── VDfiles
    └── nome_do_usuario
        ├── APO
        │   └── 2mu8 // Nome do arquivo .pdb submetido
        │       └── 2023-07-28T11:43:45 // Data da geração
        │           ├── figures // Figuras geradas automaticamente para análise
        │           │   ├── 2mu8_gyrate.png
        │           │   ├── 2mu8_potentialcg.png
        │           │   ├── 2mu8_potentialsd.png
        │           │   └── ...
        │           ├── run // Arquivos relacionados a execução da simulação
        │           │   ├── logs // Logs da execução (Apenas GROMACS)
        │           │   │   └── gmx.log
        │           │   ├── 2mu8.pdb
        │           │   ├── 2mu8.gro
        │           │   ├── 2mu8_potentialcg.xvg
        │           │   ├── 2mu8_potentialsd.xvg
        │           │   └── ...
        │           ├── celery_id
        │           ├── commands.txt
        │           ├── status
        │           └── steps.txt
        ├── ACPYPE // Pasta com mesma estrutura acima
        ├── PRODRG // Pasta com mesma estrutura acima
        └── dynamics.list // Lista com todas as simulações em disco
```

<!--
- Simulações geram arquivos, muitos arquivos, que são utilizados para a analise dos resultados, geração de gráficos através do `grace` e muitas outras coisas, e esses arquivos precisam ser salvos, de uma forma que seja possível acessá-los posteriormente.

- ACPYPE e PRODRG possuem alguns arquivos a mais, porém a estrutura segue a mesma

- APO Proteína é uma proteína sozinha, enquanto ACPYPE e PRODRG são proteínas complexadas, em suma, proteína + ligante.

- ACPYPE e PRODRG são métodos diferentes da preparação do ligante para a proteína.
